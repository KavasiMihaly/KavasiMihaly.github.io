A role of a Power BI Tenant Administrator is complex especially at larger organizations.

Tasks could involve:

  • Governance: Establish guidelines and policies to support enterprise BI and managed self-service BI.

  • User Empowerment: Facilitate internal processes and systems to empower users while adhering to organizational regulations.

  • Adoption: Promote broader organizational adoption through effective governance and data management practices.

  • System Oversight: Activities associated with system administration ensure a positive experience for content consumers and creators.

In order to perform these tasks a lot of insights needed about the Power BI tenant, including performance monitoring, activity tracking, and audit information.

Let’s dive into the key APIs, services and built-in features that can help you achieve these goals:

Standard Audit information

Power BI Rest APIs provide rich metadata about all the assets available in the Power BI Tenant.

In order access this information you need to be a Power BI Administrator or you need to have a Service Pricipal which have the right to access admin apis.

Previously you needed to access many different API endpoint to built out a holistic picture about the entire content in your tenant but no you can use the scanner API endpoints to query all the relevant information.

This is an asynchronous operation:

  • Use the PostWorkspaceInfo call to request data about a set of workspaces

  • Store the ID from the response

  • Query the GetScanStatus using the ID to know if the scan is ready

  • Run the GetScanResult using the ID to get the data.

Purview

Azure Purview is a comprehensive data governance solution offered by Microsoft. It enables organizations to discover, understand, and manage their data assets across various sources. With Purview, you can create a unified catalog of your data, including on-premises, multi-cloud, and SaaS data. The platform provides features such as data lineage, data classification, and data discovery, allowing data stewards and administrators to gain insights into data usage, quality, and compliance.

In addition to cataloging data, Azure Purview integrates with other Azure services, making it easier to enforce policies, track data lineage, and collaborate across teams. Whether you’re dealing with structured or unstructured data, Purview helps you maintain data quality, ensure compliance, and drive better decision-making. Overall, it’s a powerful tool for organizations seeking to establish robust data governance practices.

Power BI Integration

Purview integrates with Power BI enabling advanced data governance solutions.

Using the same scanner APIs described previously. Currently the funcionality is limited to a few data sources, but it will enable end-2-end lineage from source system through the data layers to the Power BI report.
